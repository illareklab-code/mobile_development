# Semana 8 — Examen Parcial: Exposición Sprint 1
**Curso:** Desarrollo de Sistemas Móviles | **Unidades:** U1–U2  
**Tipo:** Evaluación grupal | **Ponderación:** 30% de la nota final (N1)

---

## Evaluación

Los equipos presentan el **Sprint 1** de su proyecto del curso ante el docente y la clase.

---

## Requisitos de la Presentación

- **Duración:** 8–10 min por equipo + 5 min de preguntas.
- **Demo en vivo:** app corriendo en dispositivo físico o emulador.
- **Recorrido de código:** mostrar la arquitectura y las decisiones técnicas.
- **Historia del repositorio:** commits limpios con mensajes descriptivos.

---

## Alcance del Sprint 1 (Semanas 1–7)

| Requisito | Descripción |
|---|---|
| Diseño Figma | Mínimo 3 pantallas diseñadas con Material Design 3 |
| Proyecto Android | Kotlin + Jetpack Compose, Kotlin DSL, API 24+ |
| Arquitectura | MVVM + Clean Architecture (UseCase + Repository) |
| Inyección de dependencias | Hilt configurado y funcionando |
| Navegación | Jetpack Navigation con type-safe routes, mínimo 3 pantallas |
| API REST | Al menos 1 endpoint consumido con Retrofit + Kotlinx Serialization |
| Firebase Firestore | Lectura y escritura de datos en tiempo real |
| Git | Repositorio GitHub con historial de commits por semana |

---

## Criterios de Evaluación

| Criterio | Peso |
|---|---|
| Demo funcional sin crashes | 30% |
| Arquitectura MVVM + Clean correctamente implementada | 25% |
| Calidad de la UI (Material Design 3, accesibilidad) | 20% |
| Integración Firebase Firestore | 15% |
| Calidad del código + historial Git | 10% |

### Evaluación individual

Cada integrante será preguntado sobre su contribución específica y las decisiones técnicas que tomó. La nota puede diferir entre integrantes del mismo equipo.

---

## Checklist de Preparación

- [ ] La app compila sin errores en API 35.
- [ ] Todos los integrantes pueden explicar la arquitectura.
- [ ] El diseño Figma está disponible para mostrar.
- [ ] El repositorio de GitHub es público y tiene un README.
- [ ] El historial de Git tiene commits de cada semana (no un solo commit de todo).
- [ ] No hay claves de API hardcodeadas en el código.
- [ ] Firebase Security Rules configuradas (no en modo abierto).
